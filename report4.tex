\end{enumerate}

\section{エディタによる複数ファイルの取り扱いについて}
Mule,viどちらのエディタにもバッファと呼ばれるデータの一時的な格納場所がある。
このバッファを使うことによって複数ファイルを取り扱うことや、同じファイル・他のファイルへのコピーなどを行うことができる。各エディタの操作方法は次の通りである。
\begin{itemize}

\item Mule
\begin{itemize}
\item 新たにファイルを読み込む\\
C-x C-f と入力しファイルを指定する。
\item バッファの切り替え（読み込み済みの別のファイルに切り替える）\\
C-x b と入力し、バッファを指定する。
\item バッファの一覧表示\\
C-x C-b
\end{itemize}

\item vi
\begin{itemize}
\item vi起動時に引数に複数のファイルを指定している場合に、次のファイルに切り替える\\
:n
\item 新たにファイルを読み込む\\
:e と入力しファイルを指定する。
\item 直前のファイルに切り替える\\
:e\#
\item 起動時の最初のファイルに切り替える\\
:rew
\end{itemize}

\end{itemize}


\section{Wnnをクライアント／サーバシステムで利用する場合の利点}
Wnnが搭載されているサーバで変換処理等を行うため、ローカル（クライアント）のCPU負荷の軽減になる他、単語登録や辞書学習のデータベースを一元管理することが可能であり、ディスク等のスペースを効率的に活用することができる。

研究室では、CPU負荷の軽減がされ、ディスクスペースも効率的に活用している。さらに、どのワークステーションでログインしても登録した単語を呼び出すことができるため一元管理の利点も活用している。

よって研究室では、Wnnがクライアント／サーバシステムを採用することで得られる利点を生かしているといえる。

\section{漢字コードについて}
漢字コードはおもに２バイトで構成されており、JISコード,シフトJISコード,EUCコードなどがある。

この文字コードに関する問題としては異なる文字コードで読み書きを行うと文字化けが発生することである。

例えば、Muleでテキストを作成しJISコードとしてファイルを保存した場合に、catコマンド等で読み込もうとすると正確に読み込めず文字化けが発生する。原因は、UNIXではEUCコードを標準の文字コードとして使用しているため、JISコードのファイルは正しく読み込めないためである。

以上のことから、エディタを使う場合は使用する文字コードに注意し必要に応じて文字コードを設定しなおさなければならない。


\section{Muleとviの比較}

\begin{itemize}
\item (a)画面表示

\begin{itemize}
\item Mule\\
画面下部に、エコーエリア（ミニバッファ）とモードラインが表示され、メッセージやテキストの情報が表示される。
\item vi\\
画面下部に、空白の行が存在し必要に応じてメッセージ等が表示される。
\end{itemize}

\item (b)文字入力

\begin{itemize}
\item Mule\\
キーを打つと該当する文字が表示される。
\item vi\\
モードという概念があり、インサートモードにモードを変更してから文字入力を開始することができる。
\end{itemize}

\item (c)キー操作

\begin{itemize}
\item Mule\\
コントロールキーもしくは、メタキーと他のキーとの組み合わせで様々な操作ができる。
\item vi\\
viコマンドモードでコマンドを入力する。またviコマンドモードで'':''を入力することではexコマンドモードにモードが変更される。どちらのモードでも様々な操作をすることができる。\\
\end{itemize}

\item (d)編集機能

Mule,viどちらもほぼ同じことをすることができるが機能や操作方法が違う。\\
また文字コードが違うので保存時等には注意する必要がある。\\

\item (e)その他気づいたこと

\begin{itemize}
\item Mule\\
Muleの知識がなくてもある程度の文字入力や操作を行うことができる。しかし、使いこなすには数多くあるコマンドを覚えることが必要である。
\item vi\\
起動時にはviコマンドモードであり、エディタがモードという概念を用いられているので知識がないと文字入力すらできない。初めて使う人は戸惑うと思われる。
\end{itemize}

\end{itemize}
