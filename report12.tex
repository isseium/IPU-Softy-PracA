\section{ネチケット}
ネチケットを、検索サイトgoo\cite{goo}を用いて検索し、フリー百科辞典wikipedia\cite{wiki}のページを参照した。参照ページ内にあったネチケットについての重要なマナーを以下に示す。
\begin{itemize}
\item 機種依存文字を使用しない
\item （電子メールにおいては）半角カナを使用しない
\item マルチポストをしない(荒らしに繋がる可能性大)
\item 引用は明示して適切に行う(最悪の場合コピー＆ペーストを繰り返す荒らしと受け取られかねない)
\item フレームを誘わない、反応しない、悪質なものは管理者に報告
\item 個人情報を流出させたりプライバシーを侵害したりしない(名誉毀損罪など犯罪の他、民事訴訟にも訴えられる事がある)
\item 不必要に巨大なデータを送らない 
\end{itemize}

\section{SMTP,およびPOPの認証機能の必要性}
\label{SEC:NINSYO}
SMTP,およびPOPを郵便局と対比させると、SMTPはポストおよび配達機能であり郵便を投函するポストや窓口には不特定多数の郵便物を匿名や偽名で投函することができる。またPOPは、郵便物が配達されたポストから取り出す機能であり、配達されてポストに入った郵便物を取り出す人はその郵便物の宛先人のみである。

これらのことより、SMTPにはユーザ認証が必ずしも必要であるわけではないが、POPにはユーザ認証が必ず必要となる。

しかしながら、SMTPに認証が必要ないということはメール送信を匿名や偽名で行うことができるため、スパムメールやなりすましメールの原因となっている。現在では、この問題に対応するためSMTP認証と呼ばれる機能もある。

\section{NNTPのコマンド\cite{NNTP}}
HEADコマンドは、記事のヘッダ情報を取得するコマンドである。

NEXTコマンドは、記事を次に進めるコマンドであり、HEADコマンドと併用することで、記事のヘッダ情報を順に表示していくことが可能となる。

\section{SPAM}
SPAMとは、「受信者の意図を無視して、無差別かつ大量に一括して送信される、電子メールを主としたメッセージ」と述べられている\cite{wiki}。SPAMは、料金が安い、短時間で大量送信が可能といったことから、様々な商品、サービスの広告として不特定多数のメールアドレスへ配信されており現在問題となっている。このようになった原因は、第\ref{SEC:NINSYO}節で述べた通り、メールの送信には認証の必要がなく通常は送信制限がないためといったことが挙げられる。
