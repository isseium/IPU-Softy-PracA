\section{ワークステーションの基本構成}
ワークステーションは以下の５つの基本構成から成り立っている。

\begin{itemize}
\item ハードウェア（Hardware）\\
物理的な装置全般をハードウェアと呼ぶ。演算処理などを中心に行うCPU、情報の読み書きを行うメモリ、キーボードなどの入出力装置が代表的である。
\item オペレーティングシステム(Operating System)\\
ワークステーションで実行するプログラムの制御等を行う。いわばコンピュータ内部と利用者の仲介役である。\\
\item ライブラリ(Library)\\
ソフトウェア開発を行う際に使われる基本的な処理手順をまとめたもの。これを用いることでソフトウェア開発の効率が上がる。\\
\item コマンド(Command)\\
ワークステーションに対する命令を実行可能な形式としたもの。単純なものから、複雑なものまで数多く存在する。\\
\item 応用プログラム(Application Program)\\
利用者が開発したプログラムであり、ワークステーションにあるコマンド、ライブラリを駆使して作成される。\\
\end{itemize}

\section{文字出力の過程}

\section{ネットワーク環境を利用することで実現可能となること。}
\begin{itemize}
\item 遠隔地のプログラムやデータ、装置の利用\\
利用したいデータ等が自分の端末に存在しなくても、自分の端末に存在している感覚で利用することができる。\\
\item 信頼性の向上\\
まったく同じファイルを２つ以上のマシンに格納することにより、１台が故障などによって利用できなくなっても、もう一方のマシンのファイルを利用することが可能になる。さらにネットワーク上に複数のCPUが存在している場合、１つが故障しても他のCPUを利用することも可能である。\\
\item 負荷の分散\\
特定の処理装置への負荷が集中する場合、ネットワーク越しに他の処理装置への負荷を分散することが可能である。\\
\item 利用者間のコミュニケーション\\
地理的に離れていても、ネットワークを利用すれば離れたところにいる人と連絡を取り合ったりすることで、共同作業をすることが可能となる。\\
\end{itemize}

\section{talkの使用方法}
talkコマンドを最初に実行（送信）する利用者をA、最初に受信する利用者をBとする。\\

\begin{enumerate}
\item A、B両者のワークスペースメニューからコンソールを起動。\\
\item Aが以下のコマンドを入力。\\
talk＜Bのログイン名＞＠＜Bが利用しているホスト名＞\\
\item Bのコンソールにメッセージが表示されてから、Bが以下のコマンドを入力。\\
talk ＜Aのログイン名＞＠＜Aが利用しているホスト名＞\\
\item  接続が確立され、半角英数字および記号のみのチャット通信が可能となる。\\
Ctrlキー+Cで接続が中断され終了する。\\
\end{enumerate}

\section{コマンドとライブラリの違い}
コマンドは、端末エミュレータ等で入力するワークステーションに対する命令であり、単体で実行可能であるが、ライブラリは汎用性が高い処理手順等を再利用可能な形式としたものであり、単体では実行することはできず、プログラム開発の際に「部品」として機能を提供するものである。\\

\section{デーモンプロセスについて}
デーモンプロセスとは、システム上に常駐して動き続けユーザがいなくてもバックグラウンドで処理等を行うプロセスのことである。\\

\section{write,mesgコマンドの概要とtalkコマンドとの関係}
\begin{itemize}
\item 概要\\
\begin{tabular}{|l|l|}
\hline
コマンド名 & 概要\\
\hline
write & ログインしているユーザ端末を指定しメッセージを送信する。\\
mesg & 他ユーザからのメッセージの許可／禁止を設定する。\\
\hline
\end{tabular}

\item talkコマンドとの関係\\
talkコマンドは、リアルタイムでメッセージを交換することができるが、両者が接続を確立することが必要である。一方、writeコマンドはリアルタイムでメッセージを交換することはできないが、接続を確立しなくても一方的に送信することが可能である。\\
mesgコマンドは、talk,writeコマンドの許可や禁止（拒否）を設定することができる。\\
\end{itemize}

\section{ライブラリ /usr/lib/libc.aについて}
標準Cライブラリ。C言語等を用いた開発の際に用いられる関数のうち、基本的な関数などを集めたライブラリである。\\

\section{メールサーバ、ニュースサーバについて}
メールサーバは、電子メールの配送を行うコンピュータのことであり、ニュースサーバはニュース記事をユーザに送信したり、投稿を受け付けるコンピュータのことである。現在、ニュースサーバについては、掲示板やブログ等のサービスに押され衰退傾向にある。\\
