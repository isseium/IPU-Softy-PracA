\section{リモートログイン}
rloginコマンド、telnetコマンドのログイン手順を以下に示す。

\begin{itemize}
\item rloginのログイン手順
\begin{enumerate}
\item rloginコマンドを実行する。引数には、対象となるホスト名を入力する
\item パスワードを入力する
\item ログインが完了する
\item exitコマンドを実行すると接続を切断する
\end{enumerate}
\item telnetのログイン手順
\begin{enumerate}
\item telnetコマンドを実行する。引数には、対象となるホスト名を入力する
\item ユーザ名、パスワードを入力する
\item ログインが完了する
\item exitコマンドを実行すると接続を切断する
\end{enumerate}
\end{itemize}

実際の利用例を以下に示す。

\begin{verbatim}

この２つのコマンドの違いは、プロトコルが違うというである。
また、rloginはある設定をすることでログイン時にパスワード入力の操作を省くことが可能となる。一方、telnetはUNIX以外のOSに接続することが可能であり、他のOSへログインすることも可能である。

\begin{enumerate}

\item 指定したファイルをダウンロードする(メタキャラクタやワイルドカードを使用すると便利である）

\section{ftp,rcpコマンド}
ftpコマンドは、指定したホストにユーザ名、パスワードを入力してログインすることでファイルを送受信することが可能である。
それに対して、rcpコマンドはログイン作業をすることなく使用でき、cpコマンドと同様の書式でファイルのコピーを行うことができるが
事前にリモートホスト側でホスト名、ユーザ名を指定し、リモートログインを許可する必要がある。

rcpコマンドの利用手順を以下に示す。
\begin{enumerate}
\item リモートホスト側のホームディレクトリにリモートログインを許可するホスト名とユーザ名を記述した'.rhosts'を作成する
\item rcp [転送するファイル] [転送先ホスト名:転送先ディレクトリとファイル名] と入力し実行する
\end{enumerate}

通常は上の方法でログイン認証作業を省略することが可能となるが、岩手県立大学の環境の場合は自分のユーザ名以外のログイン認証作業を省略することはできない。原因としては、セキュリティの関係上許可しない設定になっていると思われる。

\section{イーサネットアドレス}
イーサネットアドレスは、イーサネット機器に割り当てられている識別子であり、ネットワークの構成とは無関係であるため、イーサネットアドレスからその機器の位置を特定するのは不可能である。このことから全世界のホスト同士のデータのやりとりを行うことはできない。

\section{.rhostsの安全性\cite{rhosts}}
rloginは、接続要求した発信元のIPアドレスから、ホスト名をDNSを使って得てから、.rhostsに書かれたホスト名と照合することで相手を認証し、パスワード入力を省略している。しかし、DNSホストに侵入された場合にDNSのデータベースを書き換えられると.rhostsに書かれたホスト名を名乗ることが可能となる。

\section{ネットワークに関するコマンド\cite{ping}}
\paragraph{ping}
ネットワーク上の指定したコンピュータにデータを送り，そのコンピュータから返信があるかどうかで，ネットワークの通信状態を調べる。

\section{sshについて\cite{ssh}}
ssh(Secure Shell)は、rlogin、およびtelnetといったコマンドと同様にネットワークを利用してリモートログインするコマンドである。rlogin、およびtelnetはネットワーク上にユーザ名やパスワードなどを平文で送信しており、他のコンピュータがネットワーク経路上でのぞき見できてしまうという問題点があったが、sshは暗号化されるためrlogin、およびtelnetなどよりは安全に通信をすることが可能となった。
